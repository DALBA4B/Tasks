ТЕХНИЧЕСКОЕ ЗАДАНИЕ ДЛЯ РАЗРАБОТКИ ПРИЛОЖЕНИЯ

Основная концепция
Приложение предназначено для создания, хранения, редактирования и отслеживания задач с дедлайнами, статусами и приоритетами.
Цель — удобное управление задачами, визуальное отслеживание сроков и статусов.

Структура задачи (Task)
Каждая задача содержит:

Название - минимум 1 символ, обязательно

Описание - опционально

Дедлайн - опционально, формат даты (последний день выполнения)

Приоритет - низкий, средний, высокий, по умолчанию: без приоритета

Статус задачи - обычная, в работе, выполнена, архивная

Системные даты:

дата создания (created_at) — автоматически, можно изменить вручную

дата перехода в статус "в работе" (in_work_at) — автоматически, можно изменить вручную

дата выполнения (completed_at) — автоматически, можно изменить вручную

Цветовая логика задач
3.1. Цвет по дедлайну
Условие | Цвет
Дедлайн прошёл (сегодня > дата дедлайна) | Чёрный
Дедлайн сегодня | Красный
Завтра | Оранжевый
Через 2 дня | Жёлтый
Через 3 дня | Светло-жёлтый
Через 4+ дней | Серый
Без дедлайна | Белый

3.2. Статус "В работе"

Задача в работе сохраняет цвет по дедлайну

Дополнительно отображается рамка

3.3. Возврат из архива

Задача возвращается в самый низ списка

При этом получает фиолетовый цвет

3.4. Архив

Задачи перемещаются в архив автоматически после выполнения

Подтверждение не требуется

В архиве можно редактировать все поля задачи

Цвет задачи в архиве не изменяется

Архив и возврат

Задачи можно возвращать из архива

После возврата они идут в самый низ списка

Получают фиолетовый цвет

Состояние и дедлайн пересчитываются по текущей логике цвета при необходимости

Сортировка

Порядок сортировки в текущей версии откладывается на будущее

Единственное правило: возвращённые из архива задачи всегда идут внизу списка

Дополнительно

Название задачи — обязательное поле (минимум 1 символ)

Задача может быть в работе даже если дедлайн прошёл

Дедлайн, даты создания, даты выполнения и статусы могут редактироваться вручную


Приложение на чистом HTML + CSS + JavaScript без дополнительных библиотек и фреймворков
Для хранения и синхронизации данных используется Firebase Realtime Database (подключение через CDN)
Не использовать React, Vue, Angular, Tailwind и другие фреймворки
Офлайн-режим: изменения сохраняются локально, синхронизируются автоматически при появлении интернета
Один userId = одна учётная запись на всех устройствах

Каждая задача должна иметь поле updated_at (timestamp), которое обновляется при любом изменении

Цвет задачи при возврате из архива: После возврата из архива задача всегда получает и сохраняет Фиолетовый цвет, даже если другие правила (по дедлайну, статусу) предполагают другой цвет. Фиолетовый цвет имеет наивысший приоритет над любой другой цветовой логикой.

Временное правило сортировки: На текущем этапе все задачи, возвращенные из архива, должны идти в самом низу общего списка.

Тип пользователя: Приложение является однопользовательским (для личного использования). Не требуется функционал аутентификации, регистрации или разделения задач по пользователям.

Платформа: Приложение должно быть адаптировано для использования как на ПК, так и на телефоне.

Логирование: Функция истории изменений (аудит лог) системных полей (created_at, in_work_at, completed_at) пока что не требуется.

Желательно использовать IndexedDB чем localStorage

Желательно использовать ISO-формат хранения дат
